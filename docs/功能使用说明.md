# FlowNoteMauiApp 功能使用说明

本文档面向产品、测试、实施与终端用户，说明当前版本主要功能、交互规则与注意事项。

## 1. 首页（文档管理）

### 1.1 列表与筛选

- 顶部支持筛选：全部 / PDF / 笔记 / 文件夹
- 支持关键词搜索（即时过滤）
- 支持排序菜单：
  - 按时间升序
  - 按时间降序
  - 按名称升序
  - 按名称降序

### 1.2 文档打开

- 点击文档卡片进入编辑页
- 顶部标签栏支持多文档并行打开
- 切换标签可切换当前文档上下文

## 2. 编辑页（PDF + 手写）

### 2.1 顶部工具区

- 导入：导入本地文档
- 搜索：进入 PDF 文本搜索
- 模式：输入模式切换（手写/触控/阅读）
- 设置：打开设置面板
- 缩略图：打开页面缩略图面板

### 2.2 输入模式

- 手写模式：面向笔输入
- 触控模式：单指书写、双指导航
- 阅读模式：仅阅读，不写入

### 2.3 书写工具

- 圆珠笔（默认）
- 钢笔
- 铅笔
- 记号笔
- 橡皮（像素 / 笔画 / 套索）

每个工具支持独立参数：

- 颜色
- 笔宽
- 压感强度（非橡皮）

## 3. 图层功能

- 新增图层
- 删除图层
- 切换当前图层
- 图层可见/隐藏
- 图层锁定
- 图层透明度（如 UI 提供）

## 4. 缩略图功能

### 4.1 基础能力

- 显示页面缩略图
- 点击缩略图跳转页面
- 当前页高亮

### 4.2 笔记叠加

开关：`Show Notes Layer`

- 开启：缩略图显示 PDF + 手写叠加
- 关闭：只显示 PDF 原图

### 4.3 性能说明

叠加模式会增加渲染成本，尤其在以下场景：

- 大文档（页数多）
- 笔迹量大
- 高频缩放/翻页

当前版本已做优化：

- 限制并发渲染数
- 预计算笔画包围盒
- 仅处理与页面相交的笔画
- 叠加开启时限制缩略图窗口数量

## 5. 书写区域规则

### 5.1 页面与页间距

- 书写允许跨过页间距进行连续笔画
- 提交后会进行裁剪：仅保留页面区域相关笔迹

### 5.2 PDF 两侧书写开关（新增）

设置路径：`设置 -> 页面设置 -> 允许在 PDF 两侧书写`

- 开启（默认）：左右两侧可落笔
- 关闭：左右两侧禁止落笔，只保留“页面内 + 页间距”

该开关会持久化，下次启动保持上次状态。

## 6. 清空笔记行为

- 点击清空按钮后会弹出确认弹窗
- 仅清空当前页笔迹（不是整份文档）

## 7. 设置页说明

### 7.1 页面设置

- 翻页方向（横向/纵向）
- 翻页模式
- 适配模式
- 文字选取与编辑
- 缩放跟随
- 允许在 PDF 两侧书写（新增）

### 7.2 显示设置

- 主题：浅色 / 深色 / 跟随系统
- 日期格式
- 保持屏幕常亮
- 暗黑模式颜色反转

### 7.3 语言设置

- 简体中文
- 英文

## 8. Android 调试日志解释

### 8.1 `open_from_bundles ... FastDev`

调试部署信息，通常不是异常。

### 8.2 `Skipped xxx frames`

表示 UI 线程有耗时操作，需继续做启动链路和渲染路径优化。

### 8.3 `valid TextAppearance attribute` 崩溃

属于主题链错误，需确保使用 MAUI 标准主题继承。

## 9. 测试建议清单

### 9.1 功能测试

- 模式切换可点击且不穿透
- 每种笔的颜色/粗细/压感生效
- 图层显示与锁定行为正确
- 缩略图跳页准确
- 叠加开关即时生效
- 清空当前页仅影响当前页

### 9.2 性能测试

- 大文档（100+ 页）打开耗时
- 叠加开启/关闭的帧率差异
- 连续写入+缩略图刷新的卡顿峰值

### 9.3 稳定性测试

- 快速切换文档标签
- 快速开关缩略图/图层面板
- 长时间书写后保存/恢复

## 10. 是否需要 C++/Rust

建议顺序：

1. 先做 C# 层算法和渲染优化（当前仍有空间）
2. 通过 profiler 证明确认瓶颈在可原生加速路径
3. 再评估 C++/Rust FFI

结论：现阶段**不建议立即引入**。先把渲染路径做完分层优化，通常即可满足体验目标。
